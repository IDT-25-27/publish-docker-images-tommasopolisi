# Pubblica Immagini Docker

_Impara come compilare e pubblicare immagini Docker usando GitHub Actions, passando dai comandi base all'automazione avanzata con azioni Docker specializzate._

</header>

## Benvenuto

- **A chi è rivolto**: Studenti, sviluppatori e ingegneri DevOps interessati ad automatizzare la creazione e pubblicazione di immagini Docker.
- **Cosa imparerai**: Come compilare e pubblicare immagini Docker usando GitHub Actions con versionamento automatico e gestione del ciclo di vita.
- **Cosa costruirai**: Un workflow di GitHub Actions che costruisce e pubblica automaticamente immagini Docker nel GitHub Container Registry.
- **Prerequisiti**:
  - Familiarità con i workflow di GitHub Actions al livello dell'esercizio [Hello GitHub Actions](https://github.com/skills/hello-github-actions)
  - Familiarità base con la containerizzazione e [Docker](https://docs.docker.com/get-started/docker-overview/)

- **Quanto dura**: Questo esercizio richiede meno di 1 ora per essere completato.

In questo esercizio:

1. Imposterai un ambiente di sviluppo e creerai un workflow Docker base per costruire e pubblicare sul GitHub Container Registry
1. Migliorerai il workflow con azioni Docker specializzate per prestazioni migliori
1. Implementerai il tagging dinamico per il versionamento automatico basato sul repository Git
1. Farai pratica con un ciclo di vita di sviluppo completo creando funzionalità, pull request e release con il corretto versionamento dei container

### Come iniziare questo esercizio

1.  Fai clic con il tasto destro su **[Step 0](../../actions/workflows/0-start-exercise.yml)** e aprilo in una nuova scheda.
2.  Clicca il pulsante **Run workflow**.
3.  Seleziona il branch `main` e clicca il pulsante verde **Run workflow**.
4.  Attendi circa 20 secondi e aggiorna questa pagina.
